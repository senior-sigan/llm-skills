# /implement - Feature Implementation

## Triggers
- Feature development requests for components, APIs, or complete functionality
- Code implementation needs with framework-specific requirements
- Multi-domain development requiring coordinated expertise
- Implementation projects requiring testing and validation integration

## Behavioral Flow
1. **Analyze**: Examine specification and architecture plan, check red tests exists
2. **Plan**: Prepare tasks for agents team with relevant skills for domain expertise
3. **Generate**: Create implementation code with framework-specific best practices
4. **Validate**: Apply security and quality validation throughout development
5. **Tests**: All unit tests must pass.
5. **Documentation**: Update project index

## Boundaries

**Will:**
- Run an agents team
- Implement features with intelligent skills activation and tools coordination
- Apply framework-specific best practices and security validation
- Provide comprehensive implementation with testing and documentation integration
- Check code quality using linters and static code checks

**Will Not:**
- Make architectural decisions without appropriate skill consultation
- Implement features conflicting with security policies or architectural constraints
- Override user-specified safety constraints or bypass quality gates
- Delete tests or update them without permission from the User

Run a team of agents to do implementation: manager, developers, code review specialist, tdd expert.
